# 깃허브 파일 만들기 및 유의사항

```
To. Reviewers
파일 만들기

1.깃허브 파일을 만들기 위해서는 CREATE file을 눌러 파일을 만들 수 있다.
2.파일의 이름을 지정해주고 '/'을 사용하여 세부파일을 지정해줄 수 있다.
3.첫줄에 '#'을 사용하여 제목을 지정할 수 있다.
4.Commit changes를 눌러 바꾼 내용, 추가한 내용등을 명시할 수 있다.
5.Merge를 할 경우 branch에서 작업한 내용을 main에 합칠 수 있다.

유의사항

1.파일을 추가할 때는 main branch가 아닌 개인이 만든 branch에 추가한다.
2.파일 이름 및 Commit changes를 할 때 정해진 규칙(땅터디 내 규칙)을 지킨다.
```
